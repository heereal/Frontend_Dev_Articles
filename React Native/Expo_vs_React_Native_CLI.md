# Expo vs React Native CLI

## Expo
### Expo란?
- Expo는 리액트 네이티브 애플리케이션을 만들 수 있는 프레임워크이자 플랫폼이다.
- 리액트 네이티브 프로젝트를 생성할 때 복잡한 설정이나 구성을 간소화해줌으로써 단순화된 앱 개발 경험을 제공하는 것을 목표로 한다.

<br/>

### 장점
1. **빠른 시작**: 디바이스에 따른 설정이나 네이티브 코드를 걱정할 필요 없이 빠르게 개발을 시작할 수 있다.
2. **Over-the-Air 업데이트**: 사용자들이 앱 스토어에서 새로운 버전 파일을 다운로드 받지 않고도 직접적으로 앱을 업데이트 할 수 있다.
3. **Pre-built 라이브러리들에 대한 접근성**: 미리 구축된 폭 넓은 라이브러리와 API를 제공하기 때문에  푸시 알림이나 인앱 구매, 지도와 같은 기능을 쉽게 구현할 수 있다.
4. **간소화된 빌드 과정**: 엑스포는 빌드 도구를 구성하거나 네이티브 코드를 컴파일하는 과정을 간소화한다.
5. **EAS**: 엑스포에서는 iOS용 앱을 구축하는 데 Mac이 필요하지 않다. 클라우드에서 EAS(Expo Application Services)에 코드를 구축하고 배포할 수 있으며, 앱 스토어에 직접적으로 배포까지 가능하다.
6. **네이티브 디바이스**: Expo Go 앱을 핸드폰에 설치한 후 같은 네트워크를 이용한다면 하나의 디바이스에서 안드로이드와 iOS 모두 테스트 할 수 있다.

<br/>

### 단점
1. **한정된 네이티브 모듈**: 엑스포는 특정 네이티브 모듈에 대한 접근을 제한하기 때문에 써드파티 라이브러리와 통합하거나 low-level 디바이스 기능에 접근할 때 제한이 발생할 수 있다.   
2. **빌드 시 통제력 감소**: 빌드 구성이나 커스텀 옵션에 대해 리액트 네이티브 CLI에 비해서 통제할 수 있는 부분이 적다.   
3. **앱 사이즈와 성능**: 엑스포 앱은 리액트 네이티브 CLI에 비해 큰 번들 사이즈를 가지고 있는데 이 때문에 다운로드 및 로딩 시간에 영향을 미친다. 또한 특정 성능 최적화는 엑스포에서 불가능하다.

<br/>

## React Native CLI
### 장점
1. **완전한 네이티브 모듈 지원**: React Native 생태계에서 사용할 수 있는 네이티브 모듈에 접근하고 통합할 수 있다. 따라서 써드파티 라이브러리와 디바이스 기능들을 유연하게 활용할 수 있다.
2. **커스텀 옵션**: 빌드 구성을 완전하게 통제할 수 있기 때문에 앱의 성능, 사이즈 등을 세부적으로 조정할 수 있다.
3. **생태계 호환성**: React Native CLI는 모든 커뮤니티 라이브러리 및 도구를 포함한 전체 React Native 생태계와 완벽하게 호환된다.

<br/>

### 단점
1. **높은 러닝 커브**: 리액트 네이티브 CLI는 초기 설정과 구성을 할 것이 많기 때문에 리액트 네이티브나 모바일 앱 개발이 처음인 개발자에게는 부담스러울 수 있다.
2. **오래 걸리는 개발 세팅 시간**: React Native CLI 프로젝트를 설정하려면 네이티브 종속성을 설치 및 구성해야 하는데, 이는 Expo를 시작할 때보다 더 오래 걸릴 수 있다.
3. **수동 빌드 과정**: React Native CLI를 사용하려면 빌드 도구를 수동으로 구성해야 하므로, 특히 네이티브 앱 개발에 익숙하지 않은 개발자의 경우 시간이 많이 걸리고 복잡할 수 있다.
4. **네이티브 전용 디바이스**: 앱을 안드로이드와 iOS에서 테스트하려면 해당 플랫폼에 맞는 물리적 디바이스가 필요하다. iOS의 경우 앱을 빌드하고 테스트하기 위해 Mac이 반드시 필요하다.

<br/>

## Expo와 React Native CLI 중에 무엇을 선택할까?
### Expo
- 앱 개발이 처음이거나 프로토타입을 빨리 만들고 싶은 경우
- low-level 디바이스 기능들에 대한 광범위한 접근이 필요하지 않은 경우
- 다양한 커스텀 옵션보다는 단순성과 쉬운 개발이 중요한 경우

<br/>

### React Native CLI
- 특정 네이티브 모듈에 접근해야 하거나 복잡한 써드파티 라이브러리와 통합해야 하는 경우
- 빌드 과정, 성능 최적화, 커스텀 옵션을 광범위하게 통제해야 하는 경우
- 네이티브 앱 개발을 경험해 봤거나 리액트 네이티브를 심층적으로 공부해 보고 싶은 경우
